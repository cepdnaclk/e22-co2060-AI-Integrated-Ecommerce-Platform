## 🎯 System Capabilities

### ✓ Automatic Training Pipeline
```
50+ Approved Q/A → Export → Split → Train → Evaluate
    → Compare Baseline → A/B Test → Deploy Winner
```

### ✓ 3 Retraining Triggers
- **Scheduled**: Daily at 2 AM (configurable)
- **Data-driven**: After 50+ new approvals (configurable)
- **Performance**: If accuracy drops >5% (auto-triggers)

### ✓ A/B Testing
- Route 10% traffic to new model (canary)
- Monitor for 24 hours
- Auto-promote if better, auto-rollback if worse

- Track all model versions in MongoDB

- Easy rollback to previous version

### ✓ Production Metrics
- Exact Match % (80%+ target)
- Token F1 (85%+ target)

- Response latency, error rate

## 🎓 Key Concepts

**LoRA** (Low-Rank Adapter)
- Efficient fine-tuning with 10x less memory
- Adapters are small (1-2GB instead of full 15GB model)

**SFT** (Supervised Fine-Tuning)
- Training on labeled Q/A pairs
- Standard approach for domain adaptation

**A/B Testing**
- Compare new model vs production on live traffic
- Gradual rollout (10% → 100%)
- Auto-rollback if metrics degrade

**Canary Deployment**
- Route small % to new version first
- Monitor carefully before full rollout
- Reduces risk of bad deployments
