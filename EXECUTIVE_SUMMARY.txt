   - Format: SOQR4:<24-character-hex-id>
   - Example: SOQR4:507f1f77bcf86cd799439011
   - Regex: /^SOQR4:([a-fA-F0-9]{24})$/
   - Backward compatible with 3 legacy formats

✅ QR Scan Endpoint
   - Route: POST /api/dms/shipments/scan-seller-qr
   - Protected by auth middleware + role check
   - Accepts: { "qrText": "SOQR4:..." }
   - Returns: 200 OK with trackingNumber

✅ Business Logic Validation Chain
   1. Parse QR → Extract order ID ✅
   2. Lookup order in database ✅
   3. Validate order not cancelled ✅
   4. Validate seller QR approved (KEY CHECK) ✅
   5. Validate courier company approved ✅
   6. Validate delivery branch approved ✅
   7. Create delivery order ✅
   8. Generate tracking number ✅
   9. Log audit entry ✅
   10. Return success response ✅

✅ Error Handling (all scenarios)
   - 400: Invalid/missing QR, cancelled order, not approved
   - 404: Order not found
   - 409: Already linked to different courier
   - 403: Unauthorized access
